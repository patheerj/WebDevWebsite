Benefits:
  • Human-Readable: Easy for humans to understand and write.

Local Storage

Local Storage is a web storage mechanism that allows web applications to store data locally within the user's browser. It provides a way to persist data across sessions without needing to interact with the server, making it useful for storing small amounts of data such as user preferences, settings, and simple state information.

Benefits:
  • Simplicity: Easy to use with straightforward methods for storing and retrieving data.
  • No Expiration: Unlike cookies, data in Local Storage does not expire unless explicitly removed.
  • Persistence: Data remains available across browser sessions.

Usage:

  • localStorage.setItem(key, value)
    -> saves the state (for key/value pair)

  • localStorage.getItem(key)
    -> retrieves a saved state (for key/value pair)

  • localStorage.removeItem(key)
    -> removes an entry (for key/value pair)

  • localStorage.clear
    -> clears all local storage

AJAX (Asynchronous JavaScript And XML)

AJAX (Asynchronous JavaScript and XML) is a technique used in web development to create dynamic and interactive web applications. It allows web pages to be updated asynchronously by exchanging small amounts of data with the server behind the scenes, without reloading the entire page. This results in a more responsive and faster user experience.

Benefits:
  • Improved User Experience: Web pages can be updated without full page reloads, leading to faster interactions.
  • Reduced Server Load: Only necessary data is sent and received, reducing the amount of data transferred and server load.

Fetch API

The Fetch API is a modern web API for making HTTP requests to servers from web browsers. It provides a more powerful and flexible feature set than the older XMLHttpRequest, allowing for easier and more readable asynchronous code, especially when combined with promises and async/await.

Benefits:
  • Simpler API: Easier to use than XMLHttpRequest.
  • Better Error Handling: More straightforward handling of errors and responses.
  • Streamlined Asynchronous Code: Works well with promises and async/await, resulting in cleaner and more maintainable code.
